# 各实验模型训练思路

## `run_exp_roi_nyha3class_5fold.py`：ROI 区域 NYHA 三分类五折训练实验

### 1. 实验目标

该入口脚本用于一键运行 ROI 图像上的 NYHA 三分类五折交叉验证实验。它本身不直接实现模型训练细节，而是承担三个核心职责：

1. 根据命令行参数选择 ROI 实验配置文件；
2. 在训练前做严格 preflight 检查，确保 ROI 图像、五折划分、标签映射和实验超参数一致；
3. 顺序调用五折训练脚本和汇总脚本，生成每折结果、OOF 预测和总体报告。

ROI 实验的核心思想是：在固定同一套 `splits_500` 五折划分的前提下，将输入图像从全局人脸图替换为局部 ROI 图像，例如眼部、嘴唇、脸颊、额头区域，再使用相同的 ResNet 分类训练框架评估不同局部区域对 NYHA 三分类任务的贡献。

### 2. 入口脚本与配置选择

入口脚本为：

```text
E:\projects\face2\scripts\run\run_exp_roi_nyha3class_5fold.py
```

脚本支持两种启动方式。

第一种是直接指定配置文件：

```powershell
python scripts/run/run_exp_roi_nyha3class_5fold.py --config config/train/roi/nyha_3class_eye_roi_resnet18_weightedce.yaml
```

第二种是指定 ROI 与 backbone，由脚本自动推断配置路径：

```powershell
python scripts/run/run_exp_roi_nyha3class_5fold.py --roi eye_roi --backbone resnet18
```

自动推断路径规则为：

```text
config/train/roi/nyha_3class_{roi}_{backbone}_weightedce.yaml
```

脚本内允许的 ROI 名称包括：

```text
cheek_roi, chin_roi, eye_roi, forehead_roi, lip_roi
```

允许的 backbone 包括：

```text
resnet18, resnet34, resnet50
```

但当前配置目录中实际已经存在的配置覆盖 `cheek_roi`、`eye_roi`、`forehead_roi`、`lip_roi` 的 `resnet18/resnet34/resnet50` 组合；`chin_roi` 图像目录存在，但当前未看到对应配置文件，因此不能直接通过 `--roi chin_roi --backbone ...` 成功运行，除非先补齐对应 YAML 配置。

### 3. ROI 数据输入与固定划分

ROI 图像根目录由入口脚本固定约束为：

```text
E:\projects\face2\data\processed\roi_dataset\manual_shift_data
```

每个 ROI 子目录下使用同名样本文件：

```text
{ID}.png
```

当前已存在的 ROI 子目录包括：

```text
cheek_roi
chin_roi
eye_roi
forehead_roi
lip_roi
```

每个目录当前各有 500 张 PNG 图像。训练不从 ROI 目录自动生成标签，而是严格依赖固定五折划分：

```text
E:\projects\face2\data\processed\splits_500
```

每折读取：

```text
fold_{fold}_train.csv
fold_{fold}_val.csv
```

这意味着 ROI 实验与其他 500Data 实验使用同一批样本、同一套 train/val fold，只改变输入图像区域，从而更适合做局部区域贡献对照。

### 4. Preflight 质量控制

入口脚本在正式训练前会调用 `preflight()`，这是 ROI 实验很关键的保护层。检查内容包括：

- `split_dir` 必须等于 `data/processed/splits_500`；
- `image_root` 必须位于 `data/processed/roi_dataset/manual_shift_data` 下；
- ROI 子目录名必须属于脚本允许的 ROI 名称；
- 图像文件名模板必须是 `{ID}.png`；
- 必须是 5 折、224 输入尺寸、3 分类；
- backbone 必须是 `resnet18/resnet34/resnet50`；
- 预训练权重必须为 ImageNet；
- `freeze_backbone=false`，即全网络参与微调；
- 训练超参数必须固定为受控实验设定；
- 训练增强只能包含水平翻转，不允许 color jitter 或 random crop。

脚本还会逐折检查 CSV 内容：

- 必须包含 `ID`、`patient_group_id`、`NYHA`、`label_3class`、`SEX`、`fold` 等必需列；
- `label_3class` 只能为 0/1/2；
- 验证集行的 `fold` 必须等于当前 fold；
- 训练集中不能包含当前验证 fold；
- NYHA 到三分类标签映射必须一致：`0 -> normal`，`1/2 -> mild`，`3/4 -> severe`；
- `SEX` 与 `sex_name` 必须一致；
- train 与 val 之间不能有 `patient_group_id` 泄漏；
- 每个 CSV 引用的 ROI 图片必须真实存在；
- 同一个样本不能出现在多个验证 fold；
- ROI 目录不能包含 split 中不存在的额外 PNG 文件。

因此，ROI 实验不是简单读目录训练，而是先强制保证样本划分、标签、图像文件和实验设置全部一致。

### 5. 训练脚本调用链

preflight 通过后，入口脚本会创建实验输出目录：

```text
experiments/ROI_500/{experiment_name}
```

如果同名目录已经存在，会自动追加时间戳，避免覆盖旧实验。

随后调用训练脚本：

```text

```

实际调用形式等价于：

```powershell
python scripts/train/train_nyha_3class_5fold.py --config <config_path> --output-dir <experiment_dir>
```

训练完成后再调用汇总脚本：

```text
scripts/evaluate/summarize_nyha_3class_5fold.py
```

实际调用形式等价于：

```powershell
python scripts/evaluate/summarize_nyha_3class_5fold.py --experiment-dir <experiment_dir>
```

整体调用链可以概括为：

```text
run_exp_roi_nyha3class_5fold.py
  -> 读取/推断 ROI YAML 配置

  -> train_nyha_3class_5fold.py
       -> NYHA3ClassFaceDataset
       -> build_transforms
       -> build_resnet_nyha_model
       -> compute_class_weights / build_criterion
       -> NYHA3ClassTrainer.fit

       -> mean_metrics.csv

       -> oof_metrics.csv
       -> summary_report.md
```

### 6. 数据集与图像预处理

训练脚本使用：

```text
datasets/nyha_3class_face_dataset.py
```

核心类为 `NYHA3ClassFaceDataset`。它读取固定 fold CSV，并根据 `image_root + {ID}.png` 找到 ROI 图像。与全局图像训练不同，ROI 配置中显式提供 `image_root`，因此 CSV 中即使有原始 `image_path`，训练时也会优先使用 ROI 目录下的同名 PNG。

图像读取流程为：

1. 使用 PIL 打开图像；
2. 转为 RGB；
3. 应用 torchvision transform；
4. 返回图像张量和标签/元数据。

训练和验证 transform 分别为：

```text
train: Resize(224,224) -> RandomHorizontalFlip(p=0.5) -> ToTensor -> ImageNet Normalize
val:   Resize(224,224) -> ToTensor -> ImageNet Normalize
```

归一化参数为 ImageNet 标准均值和标准差：

```text
mean = [0.485, 0.456, 0.406]
std  = [0.229, 0.224, 0.225]
```

该设计保持 ROI 图像实验与 ImageNet 预训练 ResNet 的输入分布一致，同时避免引入 color jitter、random crop 等可能改变局部医学/表型线索的增强。

### 7. 模型结构

模型构建函数位于：

```text
models/resnet_nyha_3class.py
```

核心函数为 `build_resnet_nyha_model()`。它调用 torchvision 的 ResNet：

```text
resnet18 / resnet34 / resnet50
```

如果配置中 `pretrained=imagenet`，则加载 ImageNet 预训练权重。随后替换最后的全连接层：

```text
model.fc = nn.Linear(model.fc.in_features, 3)
```

输出为 3 类未归一化 logits，对应：

```text
0: normal
1: mild
2: severe
```

所有参数都设置为 `requires_grad=True`，即不是只训练分类头，而是对整个 ResNet 进行微调。入口脚本也会强制检查 `freeze_backbone=false`。

### 8. 损失函数与类别权重

损失函数相关代码位于：

```text
losses/classification_losses.py
```

ROI 配置使用：

```text
weighted_cross_entropy
```

每个 fold 都会根据当前训练集标签重新计算类别权重：

```text
class_weight = N / (C * class_count)
```

其中 `N` 是当前 fold 训练样本数，`C=3`，`class_count` 是该 fold 内每个类别的样本数。这样做的目的是缓解 normal、mild、severe 三类样本数不平衡对交叉熵训练的影响。

如果某个 fold 的训练集缺失某一类，脚本会直接报错，因为 weighted CE 无法为缺失类别定义合理权重。

### 9. 训练策略

每个 fold 都独立训练一个模型。训练脚本会按 fold 循环：

```text
fold_0, fold_1, fold_2, fold_3, fold_4
```

每折训练流程为：

1. 读取该 fold 的 train/val CSV；
2. 构建 train_dataset 和 val_dataset；
3. 构建 DataLoader；
4. 构建新的 ResNet 模型；
5. 根据训练集标签计算 fold-specific class weights；
6. 构建 weighted cross entropy；
7. 使用 AdamW 优化；
8. 训练并在每个 epoch 后验证；
9. 用验证集 `macro_auc` 选择最佳 checkpoint；
10. 使用最佳 checkpoint 在验证集上导出预测和指标。

受控训练超参数为：

```text
batch_size = 16
epochs = 50
optimizer = AdamW
lr = 0.0001
weight_decay = 0.0001
early_stopping_patience = 10
monitor_metric = macro_auc
random_seed = 2026
num_workers = 0
pin_memory = false
use_amp = false
```

训练器位于：

```text
trainers/nyha_3class_trainer.py
```

训练过程中每个 epoch 记录：

- `train_loss`
- `val_loss`
- `val_macro_auc`
- `val_accuracy`
- `val_macro_precision`
- `val_macro_recall`
- `val_macro_f1`
- `val_balanced_accuracy`
- `lr`

如果当前 epoch 的验证 `macro_auc` 优于历史最佳，则保存：

```text
fold_x/checkpoints/best_macro_auc.pth
```

每个 epoch 也会更新：

```text
fold_x/checkpoints/last.pth
```

如果连续 `10` 个 epoch 验证 `macro_auc` 没有提升，则 early stopping。

### 10. 随机性与可复现性

训练脚本调用 `utils/experiment_utils.py` 中的 `set_random_seed()`，固定：

- Python random；
- NumPy；
- PyTorch CPU；
- PyTorch CUDA；
- cuDNN deterministic；
- cuDNN benchmark=false。

DataLoader 也使用固定 `torch.Generator()`，并按 fold 设置不同 seed：

```text
train_loader seed = 2026 + fold
val_loader seed   = 2026 + 1000 + fold
```

这保证相同配置重复运行时，随机水平翻转、shuffle 和 DataLoader worker 随机状态尽量可复现。

### 11. 单折评估

每折训练完成后，训练脚本调用：

```text
evaluators/nyha_3class_evaluator.py
```

评估器会加载：

```text
fold_x/checkpoints/best_macro_auc.pth
```

然后在该 fold 的 held-out validation set 上计算 softmax 概率与预测类别，并输出：

```text
fold_x/predictions/val_predictions.csv
fold_x/metrics/fold_metrics.csv
fold_x/metrics/confusion_matrix.csv
fold_x/metrics/confusion_matrix.png
```

`val_predictions.csv` 是逐样本预测表，包含：

- `ID`
- `patient_group_id`
- `image_path`
- `NYHA`
- `SEX`
- `label_3class`
- `pred_class`
- `prob_normal`
- `prob_mild`
- `prob_severe`
- `correct`
- `fold`

因此，它可以用于后续错误样本分析、按性别或类别分层分析，以及不同 ROI 之间的样本级预测对照。

### 12. 指标体系

指标计算位于：

```text
metrics/classification_metrics.py
```

主指标包括：

- `macro_auc`
- `accuracy`
- `macro_precision`
- `macro_recall`
- `macro_f1`
- `balanced_accuracy`

辅助指标包括：

- 各类别 one-vs-rest AUC：`auc_normal`、`auc_mild`、`auc_severe`
- 各类别 precision/recall/f1
- `severe_vs_rest_auc`
- `normal_vs_abnormal_auc`
- confusion matrix

其中 `macro_auc` 是训练过程中用于选择最佳 checkpoint 的核心指标。它比 accuracy 更适合类别不平衡的三分类任务，因为它关注多类别概率排序能力，而不是只看硬分类是否正确。

### 13. 五折汇总与 OOF 结果

训练完 5 个 fold 后，入口脚本调用：

```text
scripts/evaluate/summarize_nyha_3class_5fold.py
```

该脚本读取每折的：

```text
fold_x/metrics/fold_metrics.csv
fold_x/predictions/val_predictions.csv
```

并生成：

```text
summary/fold_metrics_all.csv
summary/mean_metrics.csv
summary/oof_predictions.csv
summary/oof_metrics.csv
summary/summary_report.md
```

其中：

- `fold_metrics_all.csv`：每折一行，便于观察 fold 间稳定性；
- `mean_metrics.csv`：主指标的五折均值和标准差；
- `oof_predictions.csv`：所有样本的 out-of-fold 预测，每个样本只来自自己作为验证集的那一折；
- `oof_metrics.csv`：基于全部 OOF 预测整体计算一次指标；
- `summary_report.md`：人可读的实验报告，包含实验设置、fold-level 结果、OOF 指标和 OOF 混淆矩阵。

需要注意，这些结果都是五折 held-out validation 结果，不是独立测试集结果。

### 14. ROI 实验的解释逻辑

ROI 实验的关键价值在于区域对照。由于划分、训练策略、backbone、损失函数和评价指标均被固定，不同 ROI 实验之间的主要差异来自输入区域本身：

```text
eye_roi / lip_roi / cheek_roi / forehead_roi / chin_roi
```

如果某个 ROI 的表现接近或超过全局人脸输入，说明该局部区域可能包含较强的 NYHA 相关视觉线索。反之，如果某个 ROI 表现明显较弱，说明该区域单独承载的信息不足，或者局部裁剪破坏了有用上下文。

该实验还可以与全局 224 图像、面部遮罩图像、黑背景图像等实验对比，用于判断模型依赖的是局部面部区域、整体面部结构，还是背景/拍摄条件等非目标因素。

### 15. 当前实现的注意事项

1. `chin_roi` 图像目录存在，但当前未看到对应 YAML 配置。若要训练下巴 ROI，需要先补齐 `nyha_3class_chin_roi_{backbone}_weightedce.yaml`。

2. `lip_roi` 配置中的 `image_root` 当前写作 `manual_shift_data//lip_roi`，双斜杠通常不影响路径解析，但从配置规范性看可以改成 `manual_shift_data/lip_roi`。

3. 入口脚本强制要求 ROI 目录不能有 split 外额外 PNG 文件。这能防止数据不一致，但如果后续只想保留额外备用图像，该检查会导致实验中断。

4. 训练输出中的 `.pth` checkpoint 体积很大，不应提交 Git；应作为实验产物或模型归档管理。

5. 当前训练使用 ImageNet 归一化和 ImageNet 预训练权重，因此 ROI 图像虽然来自医学/面部任务，但输入预处理仍与自然图像预训练模型保持一致。

6. 该训练框架没有在入口层做独立测试集评估，所有 summary 结果均来自五折验证集。若后续需要报告最终泛化性能，应额外准备独立测试集或外部验证集。

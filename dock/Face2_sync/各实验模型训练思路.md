# 各实验模型训练思路

## `run_exp_global224_imagenet_resnet18_nyha3class_5fold.py`：Global224 + ImageNet ResNet18 + NYHA 三分类 + 五折验证

### 一、实验定位

该运行程序是全局人脸 224×224 图像的 ResNet18 基线实验入口。它本身不直接实现训练循环，而是负责把一次完整实验串起来：

```text
读取固定训练配置
→ 预检五折划分、标签、图片和患者组泄漏
→ 创建实验输出目录
→ 调用五折训练脚本
→ 调用五折结果汇总脚本
```

对应入口脚本：

```text
scripts/run/run_exp_global224_imagenet_resnet18_nyha3class_5fold.py
```

默认配置文件：

```text
config/train/nyha_3class_global224_imagenet_resnet18.yaml
```

实际训练脚本：

```text
scripts/train/train_nyha_3class_5fold.py
```

结果汇总脚本：

```text
scripts/evaluate/summarize_nyha_3class_5fold.py
```

该实验使用的是单张全局人脸图像输入，不是 ROI 单区域输入，也不是多 ROI 融合输入。

### 二、当前配置对应的数据、模型和训练参数

当前 ResNet18 实验配置为：

```yaml
experiment:
  name: Global224_ImageNetResNet18_NYHA3Class_WeightedCE_5Fold
  output_dir: experiments/470Data

data:
  split_dir: data/processed/splits_470
  image_root: data/processed/global_face/global_face_parsing_hybrid_foreheadrepair_blackbg_224_png_strict/images
  image_filename_template: "{ID}.png"
  n_folds: 5
  image_size: 224
  num_classes: 3
  label_col: label_3class
  train_csv_pattern: fold_{fold}_train.csv
  val_csv_pattern: fold_{fold}_val.csv

model:
  backbone: resnet18
  pretrained: imagenet
  num_classes: 3
  freeze_backbone: false

train:
  batch_size: 16
  epochs: 50
  optimizer: AdamW
  lr: 0.0001
  weight_decay: 0.0001
  loss: weighted_cross_entropy
  early_stopping_patience: 10
  monitor_metric: macro_auc
  random_seed: 2026
  num_workers: 0
  pin_memory: false
  use_amp: false
```

输入图像目录是：

```text
data/processed/global_face/global_face_parsing_hybrid_foreheadrepair_blackbg_224_png_strict/images
```

即前面预处理程序生成的 global face parsing-hybrid forehead-repair black background 224 PNG 数据。训练时按 `ID` 拼接 `{ID}.png` 读取图片。

类别定义为 NYHA 三分类：

```text
0 = normal
1 = mild
2 = severe
```

运行脚本预检阶段要求 `NYHA` 到 `label_3class` 的映射为：

```text
NYHA 0 → label_3class 0 → normal
NYHA 1/2 → label_3class 1 → mild
NYHA 3/4 → label_3class 2 → severe
```

### 三、运行入口的职责

`run_exp_global224_imagenet_resnet18_nyha3class_5fold.py` 的核心函数是 `main()`，执行顺序如下：

```text
config_path = DEFAULT_CONFIG
→ load_config(config_path)
→ preflight(config_path, config)
→ select_output_dir(config)
→ subprocess 调用 train_nyha_3class_5fold.py
→ subprocess 调用 summarize_nyha_3class_5fold.py
→ 打印 Experiment completed
```

这个入口脚本没有设计命令行参数，默认固定使用：

```text
config/train/nyha_3class_global224_imagenet_resnet18.yaml
```

它用 `sys.executable` 调用后续脚本，意味着训练和汇总会使用启动入口脚本时的同一个 Python 解释器环境。`subprocess.run(..., check=True)` 会在训练或汇总失败时直接抛错，中断整个实验。

### 四、预检逻辑

入口脚本在真正训练前会执行 `preflight()`，这是该运行程序中很重要的一层防错逻辑。它逐折检查：

1. `data.split_dir` 是否存在。
2. `data.image_root` 是否存在。
3. 每一折的训练 CSV 和验证 CSV 是否存在。
4. CSV 是否包含必需字段：

```text
ID
patient_group_id
NYHA
label_3class
label_3class_name
SEX
sex_name
fold
```

如果没有配置 `image_root`，还会要求 CSV 中存在 `image_path`。当前配置有 `image_root`，因此图片路径由 `image_root / "{ID}.png"` 生成。

预检还会检查标签一致性：

- `label_3class` 必须是 0、1、2。
- `NYHA` 到 `label_3class` 必须符合 0→0、1/2→1、3/4→2。
- `label_3class_name` 必须分别为 `normal`、`mild`、`severe`。
- `SEX` 到 `sex_name` 必须符合 0→`female`、1→`male`。

五折划分一致性检查包括：

- 验证集 `fold` 字段必须等于当前折号。
- 训练集不能包含当前验证折的样本。
- 训练集和验证集的 `patient_group_id` 不能有交集，避免同一患者组泄漏。
- 所有验证集 ID 合并后不能重复，避免同一样本进入多个验证折。

图片完整性检查包括：

- 对训练集和验证集每一行按 `ID` 解析图片路径。
- 如果任意图片不存在，预检会列出前 20 个缺失项并终止。

因此，这个入口脚本在训练前已经防住了几类高风险问题：标签映射错误、折内样本错放、患者组泄漏、验证样本重复和图片缺失。

入口脚本通过 `select_output_dir()` 创建实验目录。当前配置下根目录为：

```text
experiments/470Data
```

实验名为：

```text
Global224_ImageNetResNet18_NYHA3Class_WeightedCE_5Fold
```

如果目标目录不存在，则使用：

```text
experiments/470Data/Global224_ImageNetResNet18_NYHA3Class_WeightedCE_5Fold
```

如果目录已经存在，则自动追加时间戳：

### 六、五折训练脚本的总体逻辑

`scripts/train/train_nyha_3class_5fold.py` 是真正执行模型训练的程序。它的主要流程是：

```text
解析参数
→ 读取 YAML 配置
→ 设置随机种子
→ 选择 cuda/cpu
→ 保存 config.yaml 到实验目录
→ 构建 train/val transforms
→ 遍历 fold 0..4
   → 构建当前折 train/val Dataset
   → 构建 DataLoader
   → 构建 ResNet18 模型
   → 根据当前训练折标签计算 class weights
   → 构建 Weighted Cross Entropy
   → 构建 AdamW optimizer
   → 调用 NYHA3ClassTrainer.fit()
   → 用 best_macro_auc.pth 调用 NYHA3ClassEvaluator.evaluate()
```

训练脚本虽然支持 `--fold`、`--epochs`、`--num-workers` 等参数，但本入口脚本没有传这些参数，因此默认训练全部 5 折，每折最多 50 epoch。

脚本开头先导入 `sklearn` 再导入 PyTorch，这是为了在 Windows 环境下让 scikit-learn 的 OpenMP runtime 先初始化，减少 PyTorch/torchvision 与 scikit-learn 的运行时冲突风险。

### 七、数据集与图像 transform

单图像数据集类为：

```text
datasets/nyha_3class_face_dataset.py
```

使用类：

```text
NYHA3ClassFaceDataset
```

每一条样本返回一个字典，主要字段包括：

```text
image
label
ID
patient_group_id
image_path
NYHA
SEX
sex_name
label_3class_name

```

图片读取方式：

```text
PIL.Image.open(image_path).convert("RGB")
```

训练和验证 transform 由 `build_transforms()` 构建：

训练集：

```text
Resize((224, 224))
RandomHorizontalFlip(p=0.5)
ToTensor()
Normalize(mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])
```

验证集：

```text
Resize((224, 224))
ToTensor()
Normalize(mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])
```

当前配置中虽然有：

```yaml
color_jitter: false
random_crop: false
```

但当前 `build_transforms()` 实际只实现了 resize、训练集水平翻转、ToTensor 和 Normalize。也就是说，本实验没有颜色扰动、随机裁剪、随机旋转等增强。

### 八、DataLoader 实现

每折都会构建两个 DataLoader：

```text
train_loader: shuffle=True
val_loader: shuffle=False
```

关键参数来自配置：

```text
batch_size = 16
num_workers = 0
pin_memory = false
```

每个 DataLoader 都会创建独立的 `torch.Generator()` 并设置种子：

```text
train_loader seed = random_seed + fold
val_loader seed = random_seed + 1000 + fold
```

worker 初始化函数为 `seed_worker()`，会把 NumPy 和 Python `random` 的 worker seed 设置为 PyTorch 派生 seed。当前 `num_workers=0`，所以 worker seed 机制暂时不明显，但保留了可扩展性。

### 九、模型构建细节

模型构建函数为：

```text
models/resnet_nyha_3class.py::build_resnet_nyha_model
```

当前配置：

```yaml
backbone: resnet18
pretrained: imagenet
num_classes: 3
freeze_backbone: false
```

训练脚本会把 `pretrained: imagenet` 解析为 `True`，然后调用 torchvision：

```text
models.resnet18(weights=ResNet18_Weights.IMAGENET1K_V1)
```

如果 torchvision 版本较旧，则回退到：

```text
models.resnet18(pretrained=True)
```

随后替换最后全连接层：

```python
model.fc = nn.Linear(model.fc.in_features, 3)
```

ResNet18 的 `fc.in_features` 通常为 512，因此最后分类头是：

```text
Linear(512 → 3)
```

程序会将所有参数设置为可训练：

```python
for parameter in model.parameters():
    parameter.requires_grad = True
```

训练脚本还明确要求：

```text
freeze_backbone = false
```

如果配置里 `freeze_backbone: true`，脚本会直接报错。这说明该基线实验不是冻结特征提取器后只训练分类头，而是对 ImageNet 预训练 ResNet18 进行全量微调。

模型输出是未归一化 logits，形状为：

```text
[batch_size, 3]
```

后续 loss 直接吃 logits；验证和评估阶段再通过 softmax 转换为三类概率。

### 十、损失函数与类别权重

损失函数来自：

```text
losses/classification_losses.py
```

当前配置：

```yaml
loss: weighted_cross_entropy
```

训练脚本会在每一折中，根据该折训练集标签单独计算类别权重：

```text
class_weight[c] = N / (num_classes * class_count[c])
```

其中：

- `N` 是当前训练折样本数。
- `num_classes = 3`。
- `class_count[c]` 是当前训练折中类别 `c` 的样本数。

如果某一训练折缺少任意类别，`compute_class_weights()` 会直接报错，因为 weighted cross entropy 在缺类情况下没有合理权重。

当前 loss 最终是：

```python
nn.CrossEntropyLoss(weight=class_weights)
```

也就是带类别权重的交叉熵。它直接接收模型 logits 和整数标签，不需要提前 softmax。

训练脚本也兼容 label smoothing 形式的 weighted soft cross entropy，但当前 ResNet18 配置没有启用独立 `loss:` 字典，也没有启用 label smoothing。因此本实验是标准 Weighted CE。

### 十一、优化器、学习率和早停

优化器固定要求为 AdamW。当前配置：

```yaml
optimizer: AdamW
lr: 0.0001
weight_decay: 0.0001
```

训练脚本会检查优化器名称，如果不是 AdamW 会直接报错。因此该 baseline 不支持从配置切换到 SGD、Adam 或其他优化器，除非修改脚本。

当前没有学习率调度器，训练全程使用固定学习率。

早停和选模指标固定为：

```text
monitor_metric = macro_auc
early_stopping_patience = 10
```

训练脚本会要求 `monitor_metric` 必须是 `macro_auc`。每个 epoch 验证后，如果 `val_macro_auc` 高于历史最佳值，则保存：

```text
fold_k/checkpoints/best_macro_auc.pth
```

每个 epoch 都会保存：

```text
fold_k/checkpoints/last.pth
```

如果连续 10 个 epoch 没有提升 `val_macro_auc`，当前 fold 提前停止。最多训练 50 epoch。

如果某折所有 epoch 的 macro AUC 都是 NaN，程序会把 `last.pth` 复制为 `best_macro_auc.pth`，以便后续评估流程仍能生成结果。但这种情况说明该折验证集中至少有类别缺失或 AUC 无法定义，需要单独检查。

### 十二、Trainer 的训练循环

单折训练器为：

```text
trainers/nyha_3class_trainer.py::NYHA3ClassTrainer
```

训练阶段 `_train_epoch()`：

```text
model.train()
→ 遍历 train_loader
→ image/label 移到 device
→ optimizer.zero_grad(set_to_none=True)
→ logits = model(images)
→ loss = criterion(logits, labels)
→ backward
→ optimizer step
→ 统计样本加权平均 train_loss
```

验证阶段 `_validate()`：

```text
model.eval()
→ torch.no_grad()
→ 遍历 val_loader
→ logits = model(images)
→ loss = criterion(logits, labels)
→ probabilities = softmax(logits, dim=1)
→ 汇总 labels 和 probabilities
→ compute_classification_metrics()
```

当前配置 `use_amp: false`，因此 AMP 混合精度不会启用。代码中保留了 AMP 逻辑，如果以后在 CUDA 上设为 true，会使用 `torch.amp.GradScaler` 和 autocast。

每折训练期间会持续写入：

```text
fold_k/logs/train_log.csv
```

其中包括：

```text
epoch
train_loss
val_loss
val_macro_auc
val_accuracy
val_macro_precision
val_macro_recall
val_macro_f1
val_balanced_accuracy
lr
```

训练结束后会生成曲线图：

```text
fold_k/curves/loss_curve.png
fold_k/curves/metrics_curve.png
```

### 十三、评价指标实现

指标函数为：

```text
metrics/classification_metrics.py::compute_classification_metrics
```

输入是：

```text
y_true: [N]
y_prob: [N, 3]
```

函数会先检查：

- 标签是一维。
- 概率形状是 `[N, 3]`。
- 概率中没有 NaN/Inf。
- 概率范围在 `[0, 1]`。
- 每行概率和约等于 1。
- 标签只包含 0、1、2。

主要指标包括：

- `accuracy`
- `macro_precision`
- `macro_recall`
- `macro_f1`
- `balanced_accuracy`
- `macro_auc`

辅助指标包括：

- `auc_normal`
- `auc_mild`
- `auc_severe`
- `precision_normal/mild/severe`
- `recall_normal/mild/severe`
- `f1_normal/mild/severe`
- `severe_vs_rest_auc`
- `normal_vs_abnormal_auc`
- `confusion_matrix`

其中 `macro_auc` 使用 sklearn 的 one-vs-rest 多分类 ROC AUC：

```python
roc_auc_score(true, prob, labels=[0, 1, 2], multi_class="ovr", average="macro")
```

如果某个 one-vs-rest 二分类在验证集中只有单一类别，相关 AUC 会返回 NaN 并记录 warning。

### 十五、五折汇总逻辑

入口脚本在训练脚本完成后，会调用：

```text
scripts/evaluate/summarize_nyha_3class_5fold.py
```

该脚本读取实验目录中的：

```text
fold_0/metrics/fold_metrics.csv
fold_1/metrics/fold_metrics.csv
...
fold_4/metrics/fold_metrics.csv

fold_0/predictions/val_predictions.csv
...
fold_4/predictions/val_predictions.csv
```

汇总输出到：

```text
summary/
```

主要文件包括：

```text
summary/fold_metrics_all.csv
summary/mean_metrics.csv
summary/oof_predictions.csv
summary/oof_metrics.csv
summary/summary_report.md
```

其中：

- `fold_metrics_all.csv`：5 个 fold 各自的验证集指标。
- `mean_metrics.csv`：主要指标的五折 mean/std。
- `oof_predictions.csv`：把 5 个验证折预测合并成 out-of-fold 预测，每个样本应该只出现一次。
- `oof_metrics.csv`：基于全部 out-of-fold 预测重新计算整体指标。
- `summary_report.md`：Markdown 格式实验报告，记录模型、数据、训练设置、五折指标、OOF 指标和混淆矩阵。

汇总脚本会检查 OOF 中是否存在重复 ID。如果有重复，说明五折验证集不互斥，会直接报错。

需要注意：这里的 OOF 指标来自五折交叉验证的 held-out validation predictions，不是独立测试集指标。

### 十七、该实验的关键设计取舍

1. 使用固定五折划分，而不是训练时随机划分。这样每个实验之间可比较性更强。
2. 使用 `patient_group_id` 做泄漏检查，避免同一患者组同时出现在训练集和验证集。
3. 使用 ImageNet 预训练 ResNet18，全模型微调，而不是从头训练或冻结 backbone。
4. 使用 Weighted Cross Entropy，缓解 NYHA 三分类类别不平衡。
5. 使用 macro AUC 作为早停和最佳模型选择指标，比单纯 accuracy 更重视类别不平衡下的整体排序能力。
6. 数据增强很克制：只有训练集水平翻转，没有颜色扰动、随机裁剪和复杂几何增强。
7. 每折单独计算 class weights，因此权重随训练折类别分布变化。
8. 每折训练完成后立即用最佳 checkpoint 导出验证集预测，保证汇总阶段不再依赖训练过程中的内存状态。
9. 汇总阶段同时给出 fold mean/std 和 OOF 指标。前者反映折间波动，后者反映所有 held-out 样本合并后的整体表现。

### 十八、需要注意的实现细节和潜在限制

1. 入口脚本没有命令行参数，不能直接通过该脚本切换配置；如需换 backbone、数据目录或训练参数，需要改默认配置或改脚本。
2. 当前训练 transform 没有实现 `color_jitter` 和 `random_crop`，即使配置里出现这两个字段，也不会实际生效。
3. 当前没有学习率调度器，训练过程固定学习率。
4. `num_workers=0` 更稳，但数据读取可能较慢；如果后续改大，需要确认 Windows 环境下多进程 DataLoader 是否稳定。
5. `pin_memory=false`，即使使用 CUDA 也不会启用 pinned memory；这更保守，但可能牺牲部分数据传输效率。
6. `use_amp=false`，当前不使用混合精度；训练更稳定但显存和速度不是最优。
7. 预检只检查图片是否存在，不检查图片是否能被 PIL 正常解码；解码失败会在 Dataset `__getitem__` 阶段报错。
8. macro AUC 如果因为验证集类别缺失而为 NaN，会影响早停和选模。虽然五折构建通常应避免这种情况，但仍需关注日志 warning。
9. 结果汇总明确是 held-out validation / OOF，不等价于独立外部测试集。
10. 使用黑背景预处理图像时，模型看到的是统一黑背景人脸区域；如果将来换成均值背景、原图裁剪或 ROI 数据，实验结果不能直接混同，需要在实验名中区分。
